# Contributor & agent rules

Rules for anyone (human or AI) editing this repository. Follow them on every change.

## Hard rules

### 1. No em-dashes or en-dashes. Use hyphens (minus) instead.

This is a hard, non-negotiable rule. It applies to **all** content:

- The website (HTML pages, page bodies in `page_data.py`, the SHELL template in
  `build_pages.py`, `index.html`, JS in `assets/`).
- The generated Google Docs (the DOCX pipeline in `automation/`).
- File names.
- Documentation (`README.md`, `automation/SETUP.md`, etc.).

Specifically, never use any of these:

| Forbidden                          | Use instead |
| ---------------------------------- | ----------- |
| `-` em-dash (U+2014)               | `-` hyphen  |
| `-` en-dash (U+2013)               | `-` hyphen  |
| `&mdash;` / `&ndash;` HTML entity  | `-` hyphen  |
| `\u2014` / `\u2013` escape         | `-` hyphen  |

Spacing: a spaced em-dash (` - `) becomes a spaced hyphen (` - `); an unspaced
dash in a numeric range (`1-10`) becomes a plain hyphen with no spaces.

A one-shot converter that enforces this exists at
`automation/convert_dashes.py` - run it from the repo root if dashes ever creep
back in:

```
python3 automation/convert_dashes.py
python3 build_pages.py            # regenerate pages/*.html from page_data.py
python3 build_search_index.py     # regenerate search-index.json
```

Verify with:

```
grep -rn "—\|–" --include="*.html" --include="*.py" --include="*.js" \
  --include="*.json" --include="*.md" . | grep -v convert_dashes.py
```

(That command should print nothing. `convert_dashes.py` itself intentionally
contains the characters because it documents them.)

## Build / regeneration notes

- `pages/*.html` are **generated** from `page_data.py` + the SHELL template in
  `build_pages.py`. Edit the source, then run `python3 build_pages.py`. Never
  hand-edit `pages/*.html` directly - your edits will be overwritten.
- `search-index.json` is generated by `build_search_index.py`.
- The Google Docs sync (`automation/`) renders the site's CSS/HTML diagrams to
  PNGs and builds a high-fidelity DOCX per page, which Drive converts to a Google
  Doc. See `automation/SETUP.md`.
